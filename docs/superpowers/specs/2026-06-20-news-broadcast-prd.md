# 新闻播报工具 — PRD & 用户故事地图

> **项目名称**：NewsCast（暂定）
> **版本**：v0.1 MVP
> **文档日期**：2026-06-20
> **作者**：Richelleshi + Claude Code

---

## 1. 产品愿景

一个部署在网页端的个性化新闻播报工具。根据用户设定的兴趣范围，自动搜集权威媒体最新新闻，整理为两种消费模式：**双人对话播客** 和 **卡片式滑动浏览**。核心特色是对标豆包陪伴模式 — 新闻内容以自然语言输出，支持语音/文字打断和追问。

**目标用户**：个人 + 小圈子（朋友/家人）
**使用场景**：通勤途中（移动端为主）、桌面端查阅
**核心差异化**：AI 自然语言交互 + 播客陪伴感，而非冷冰冰的新闻列表

## 3. 功能需求规格 (PRD)

### 3.0 v1.1 体验调整（2026-06-23）

本轮调整的核心目标：产品必须从“打开后现生成内容的 Demo”升级为“打开即消费、可自然语音追问的新闻陪伴工具”。以下要求优先级高于早期 MVP 描述。

| 模块 | 当前问题 | v1.1 调整要求 | 优先级 |
|------|----------|---------------|--------|
| 新闻详情 | 点击卡片后仍然只是摘要，和外层卡片信息重复 | 详情页必须展示比卡片更完整的信息：事件背景、关键事实、影响分析、时间线/相关方、可信来源链接、原文摘录或抓取正文摘要 | P0 |
| 新闻卡片 | 卡片摘要和详情信息层级不清 | 卡片只放“标题 + 80-120字摘要 + 来源/热度”；详情页承载深度解读，形成明确的信息递进 | P0 |
| 追问 FAB | 不能贴边隐藏，拖动不跟手 | FAB 必须支持自由拖动、松手吸附左右边、贴边半隐藏；拖动时跟手无明显延迟；点击和拖动不能误触 | P0 |
| 新闻/播客生成 | 用户打开页面时仍在等待抓取、AI 富化和生成 | 新闻抓取、AI 解读、播客脚本、TTS 音频必须在后台定时预生成；前台只读取最新快照和音频，打开即可看/听 | P0 |
| 语音追问 | 需要手动点按，缺少连续聆听和自动回话 | 对标豆包：支持连续监听用户说话，自动 ASR，自动暂停播客，基于上下文 + Web Search 生成回答，文字流式显示，并自动 TTS 播放回答 | P0 |
| TTS 自然度 | 默认 Edge TTS 机械感明显，不适合陪伴式播客 | Edge TTS 只能作为兜底；主路径应切换到更自然的 TTS：优先 MiMo TTS API，备选 OpenAI TTS，本地方案评估 CosyVoice / IndexTTS / F5-TTS | P0 |

#### 3.0.1 新闻详情信息层级

详情页必须至少包含：
- 事件一句话结论：比标题更明确地回答“发生了什么”。
- 关键事实：3-5 条要点，包含人物/机构/时间/地点/数字。
- 背景解释：说明为什么这条新闻重要，以及和用户兴趣的关系。
- 影响分析：短期影响、可能后续、值得继续关注的点。
- 来源与可信度：展示原始链接、来源列表、发布时间；若仅来自热榜标题，必须标记“信息不足”。
- 原文/网页正文摘要：后台抓取原文正文或搜索结果后生成，不允许只复用卡片摘要。

卡片摘要和详情内容的字数建议：
- 卡片摘要：80-120 字，适合快速扫读。
- 详情深度解读：600-1000 字，分段展示，适合点开后认真阅读。
- 语音播报文本：120-200 字，独立于详情解读，专门为 TTS 口语化生成。

#### 3.0.2 后台预生成架构

前台接口不能再承担重生成职责。推荐生成链路：

```
定时任务（06:00 / 17:00 / 21:00，可配置）
  ├─ 抓取新闻源和热榜
  ├─ 抓取原文正文或搜索补充材料
  ├─ AI 去重、可信度判断、主题排序
  ├─ 生成卡片摘要、详情深度解读、播报文本
  ├─ 生成当期双主播播客脚本
  ├─ TTS 合成完整播客音频和关键片段音频
  ├─ 写入 data/current/news.json、data/current/podcast.json、audio/*.mp3
  └─ 前端打开时只读取 current 快照
```

前端加载策略：
- 首屏优先读 `current` 快照；存在缓存时不显示“正在抓取最新新闻”。
- 手动刷新只触发后台任务或读取下一份快照，不阻塞当前阅读/收听。
- 接口返回必须包含 `generated_at`、`expires_at`、`source_count`、`is_stale`，让用户知道内容新鲜度。

#### 3.0.3 语音追问体验

目标体验：用户无需手动组织完整输入，系统持续感知“用户是否在问问题”。

语音链路：
1. 进入播客或新闻详情后，用户可开启“连续聆听”。
2. 检测到用户说话时，播客自动 ducking 或暂停，开始 ASR。
3. ASR 获得稳定语句后自动判断是否为问题；非问题不打断主流程。
4. 问题发送到 `/api/ask`，上下文包含当前新闻详情、播客当前片段、最近对话历史。
5. 后端优先抓取原文；不足时调用 Web Search；回答必须给出“基于哪些来源/哪些事实”的内部依据。
6. 前端流式显示文本，同时 TTS 合成并播放语音回答。
7. 用户继续说话则进入多轮追问；用户沉默或说“继续播放”时恢复播客。

兼容策略：
- Web Speech API 可作为 MVP，但 iOS/部分浏览器不稳定；后续应预留服务端 ASR。
- 连续监听必须有明显状态提示和一键关闭，避免隐私不确定感。

### 3.1 新闻采集引擎

#### 3.1.1 新闻源体系（5层50+源）

| 层级 | 类型 | 代表来源 | 获取方式 |
|------|------|---------|---------|
| L1 权威媒体 | 官方RSS/API | 新华网、人民网、CCTV、环球网、参考消息、中国日报、光明日报、经济日报 | RSS / 网页抓取 |
| L2 品质媒体 | RSS/Web | 财新、澎湃新闻、界面新闻、南方周末、新京报、第一财经、21世纪经济报道 | RSS / API |
| L3 垂直科技 | RSS/API | 36氪、虎嗅、钛媒体、极客公园、InfoQ、机器之心、量子位、雷锋网 | RSS |
| L4 社交热度 | 热搜API | 微博热搜、知乎热榜、百度热搜、今日头条、抖音热点、B站热门 | API / 聚合 |
| L5 国际视野 | RSS/API | Reuters、BBC中文、FT中文、联合早报、日经中文、端传媒 | RSS |

**借鉴来源**：Horizon 多源AI交叉打分 + TrendRadar 35+平台热搜监控

**真实性保障机制**：
1. 同一新闻事件必须≥2个独立来源交叉验证才采纳
2. L4社交热度只能作为热度加权因子，不能作为唯一来源
3. 每条新闻保留原始链接，用户可跳转验证
4. DeepSeek 在汇总时被要求标注信息来源

#### 3.1.2 兴趣配置

用户可选择：
- 12个预设领域标签：时政要闻、科技互联网、财经商业、体育赛事、文娱影视、国际新闻、科学探索、医疗健康、教育学术、房产家居、汽车出行、环保气候
- 自定义关键词（自由输入，支持多个）
- 兴趣标签 + 关键词作为 DeepSeek 筛选排序的 prompt 输入

#### 3.1.3 新闻量设计

- 每期（2小时窗口）精选 **8-12条** 核心新闻
- 每条新闻：标题 + 200字口语化摘要 + 来源链接 + 发布时间
- 播客脚本约 **3000-4000字**（对应15-20分钟双人对话）
- 每期播客选出 **5-8条** 最值得讨论的新闻深入展开

### 3.2 播客生成

#### 3.2.1 播客脚本

| 维度 | 规格 |
|------|------|
| 长度 | 3000-4000字，15-20分钟 |
| 格式 | 双人对话（主持人A + 主持人B） |
| 深度 | 5-8条新闻，每条讨论2-4分钟 |
| 风格 | 用户可选：轻松聊天 / 专业播报 / 辩论 / 问答 |

**DeepSeek Prompt 策略**：
```
System: 你是一位资深播客内容策划。根据以下新闻列表，生成一段{风格}的双人对话播客脚本。
要求：
- 主持A（晓晓，女性）和主持B（云扬，男性）交替发言
- 口语化、自然，像朋友聊天一样
- 每条新闻讨论2-4分钟，有观点、有背景、有延展
- 总时长控制在15-20分钟
- 标注每个段落的情绪倾向（可选项：轻松/严肃/兴奋/沉思）
- 新闻信息必须真实，不添油加醋
```

#### 3.2.2 TTS 合成

| 维度 | 规格 |
|------|------|
| 主引擎 | 小米 MiMo-V2.5-TTS API（待确认正式文档、计费、音色 ID） |
| 备选云引擎 | OpenAI `gpt-4o-mini-tts` 或同级自然语音 API |
| 本地部署候选 | CosyVoice / IndexTTS / F5-TTS，优先评估中文自然度、推理成本、流式能力、部署复杂度 |
| 兜底引擎 | Edge TTS，仅用于无 Key 或离线调试，不作为正式体验 |
| 音色A（女） | 温暖、亲和、解释型女声 |
| 音色B（男） | 明亮、自然、轻松评论型男声 |
| 情绪控制 | 根据脚本标注的情绪标签，使用自然语言描述控制 |
| 播客细节 | 支持 [吸气] [轻笑] 等标签增强真实感 |
| 费用 | 免费/低成本优先；若本地部署维护成本高，则接受 MiMo 的低价 API 方案 |
| 输出格式 | MP3, 64kbps mono, 约7-10MB/期 |

**TTS 选型原则**：
1. 自然度优先于免费；机械感会直接破坏“陪伴感”核心体验。
2. 播客长音频必须后台预生成，避免用户端等待。
3. 追问回答可实时生成短音频，但要缓存相同文本，降低重复成本。
4. 云 TTS 作为短期上线方案，本地 TTS 作为成本可控和隐私增强的中长期方案。
5. 每个候选 TTS 需要用同一组中文测试文本做 A/B：新闻播报、轻松评论、追问回答、数字/英文混读、长句停顿。

#### 3.2.3 更新调度

| 维度 | 规格 |
|------|------|
| 默认时间点 | **8:00** / **18:00** / **22:00**（内容在此时间前生成完毕） |
| 触发机制 | GitHub Actions 在设定的时间点前10分钟触发 |
| 用户自定义 | 可增减时间点，最少保留1个 |
| 加载体验 | 异步预生成 → 用户打开页面直接加载已生成内容，零等待 |

### 3.3 前端交互

#### 3.3.1 两种浏览模式

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| 🃏 **卡片模式** | 新闻以卡片形式展示，Tinder式滑动切换，点击进入详情 | 通勤路上单手操作 |
| 🎧 **播客模式** | 音频播放器 + 实时字幕滚动，沉浸式收听 | 开车/走路/做家务 |

#### 3.3.2 卡片模式详细交互

- 垂直/水平滑动切换新闻
- 每张卡片：封面图（如果有）+ 标题 + 摘要（前100字）+ 来源标识
- 点击展开：完整摘要 + 原始链接（跳转验证）
- 左右滑 → 下一篇，右滑动 → 上一篇
- 顶部显示当前期数 + 更新时间
- 支持分享单条新闻链接

#### 3.3.3 播客模式详细交互

- 底部播放控制栏：播放/暂停、快进15s、快退15s、进度条
- 实时高亮字幕（当前正在说的那句话高亮）
- 播放速度：0.75x / 1x / 1.25x / 1.5x / 2x
- 支持后台播放（锁屏继续）
- 显示正在讨论的新闻标题

#### 3.3.4 陪伴交互（打断+追问）

**语音模式**（混合模式）：
1. 用户说话 → 浏览器 Web Speech API 检测到语音活动
2. 播客自动暂停，记录暂停时间戳
3. 语音转文字 → 发送到 Vercel API → DeepSeek 处理
4. DeepSeek 流式返回文字 → 屏幕显示 + MiMo TTS 语音播报
5. 用户可继续追问，或说"继续播/好了"恢复播放
6. 从暂停时间戳处无缝恢复播客

**文字模式**：
1. 用户点击输入框或开始打字 → 播客暂停
2. 输入文字 → 发送到 API → DeepSeek 流式返回
3. 文字显示在对话气泡中
4. 继续追问或恢复播放

**上下文策略**：
- 每次追问注入：当前新闻全文 + 播客脚本中被讨论的段落 + 最近10轮对话历史
- System Prompt 设定为"知识渊博、友好的新闻解说助手"
- 回答控制在口语化、1-2分钟内讲清楚

#### 3.3.5 响应式设计

| 断点 | 设备 | 布局 |
|------|------|------|
| < 768px | 手机 | 单列、全屏卡片、底部Tab导航 |
| 768-1024px | 平板 | 两列、侧边栏可选 |

### 3.4 数据存储

| 存储内容 | 位置 | 保留策略 |

### 4.3 数据流

```
GitHub Actions (每8/18/22点触发)
    │
    ├─ ① 读取 news-sources.yaml → 遍历50+新闻源
    ├─ ② 爬取RSS/API → 原始新闻列表
    ├─ ③ DeepSeek API: 去重 → 交叉验证 → 按兴趣筛选 → 排序
    ├─ ④ DeepSeek API: 生成摘要 → 生成播客脚本
    ├─ ⑤ MiMo TTS API: 合成女声音轨 → 合成男声音轨 → 合并MP3
    ├─ ⑥ 写入 data/news/YYYY-MM-DD-HH.json
    ├─ ⑦ 写入 data/podcast/YYYY-MM-DD-HH.json + podcast.mp3
    ├─ ⑧ 运行清理脚本（删除>7天数据，压缩3-7天数据）
    └─ ⑨ git commit + git push → 触发Vercel自动部署

Vercel (前端 + 实时API)
    │
    ├─ 静态页面 → 从 data/ 读取最新新闻 → 渲染卡片+播客播放器
    ├─ /api/ask → 接收用户追问 → DeepSeek 流式返回
    └─ /api/tts → 接收文本 → MiMo TTS → 返回音频
```

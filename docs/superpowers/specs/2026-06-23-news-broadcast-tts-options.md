# NewsCast TTS 选型记录

**日期**：2026-06-23
**目标**：替换当前机械感明显的 Edge TTS，让新闻播客和追问回答接近自然陪伴式语音体验。

---

## 1. 结论

短期建议：
- 主路径优先接入 **小米 MiMo TTS API**，前提是拿到正式 API 文档、价格、鉴权方式和音色 ID。
- 若 MiMo 接入不稳定，使用 **OpenAI `gpt-4o-mini-tts`** 作为云端备选。
- **Edge TTS 只保留为兜底**，不作为正式体验。

中长期建议：
- 本地部署优先评估 **CosyVoice 300M / CosyVoice2 0.5B** 和 **F5-TTS 0.3B**，因为模型体量相对适合 M1 尝试。
- **IndexTTS2 1.5B** 自然度/情感很有吸引力，但对 M1 更吃力，优先作为服务器或更高内存 Mac 的候选。
- 本地 TTS 更适合后台预生成播客，不建议一开始承担实时追问，除非能接受明显等待或单独部署推理服务。

---

## 2. 候选方案（按自然度/免费/M1 可行性）

| 方案 | 自然度预期 | 是否免费 | 模型大小/体量 | M1 可行性 | 风险/成本 | 建议 |
|------|------------|----------|----------------|-----------|-----------|------|
| MiMo TTS | 高，中文拟人化潜力强 | 可能低价/试用，需确认 | 云 API，无本地模型 | M1 无压力，只调 API | 需要正式 API 文档、价格、音色 ID | 短期主路径 |
| OpenAI `gpt-4o-mini-tts` | 中高，语气可控 | 不免费 | 云 API，无本地模型 | M1 无压力，只调 API | 中文本土播客感需试听；持续 API 成本 | 云端备选/对照组 |
| CosyVoice 300M | 中高 | 开源权重，推理免费 | 约 0.3B，磁盘通常数 GB 级 | M1 可尝试，适合离线预生成；实时性需实测 | 环境复杂，依赖 PyTorch/音频库 | 本地优先候选 |
| CosyVoice2 / 3 0.5B | 高 | 开源权重，推理免费 | 约 0.5B，磁盘通常数 GB 级 | M1 16GB 更稳；M1 8GB 可能吃紧或慢 | 部署较重，长文本速度需实测 | 本地重点候选 |
| F5-TTS 0.3B | 中高 | 代码 MIT；预训练模型需看许可，常见为非商用限制 | 约 0.3B | 官方支持 Apple Silicon PyTorch；M1 可尝试 | 需要参考音频；中文新闻播报自然度需 A/B | 本地备选 |
| IndexTTS2 1.5B | 高，情绪和音色控制强 | 开源权重，推理免费；商用需确认 | 约 1.5B，磁盘/内存占用更大 | M1 基础款不推荐实时；16GB 也可能慢，适合服务器或高配 Mac | 环境重，CUDA/FP16优化更偏向独显 | 深度评估，不作为第一落地 |
| Edge TTS | 低到中 | 免费 | 云/系统服务，无本地模型 | M1 无压力 | 机械感明显，不符合陪伴式产品 | 只兜底 |

> M1 判断口径：这里默认是普通 M1 Mac，不假设独立 NVIDIA GPU。若是 8GB 内存，本地大模型 TTS 只建议轻量试验；若是 16GB 内存，可以优先试 CosyVoice 300M/F5-TTS，播客采用后台慢速预生成。

## 3. A/B 测试文本

每个候选引擎必须使用同一组文本试听：

1. 新闻播报：包含日期、机构、数字、英文缩写。
2. 双主播评论：轻松对话、自然转场、短句停顿。
3. 追问回答：80-150 字，像朋友解释而不是读稿。
4. 长句压力测试：连续 300-500 字，观察断句、喘息感和重复问题。
5. 中英混读：AI、OpenAI、GDP、A股、2026 年等混合内容。

评分维度：
- 自然度
- 中文咬字
- 情绪/语气控制
- 长文本稳定性
- 首包延迟
- 每分钟成本
- 部署和维护复杂度

---

## 4. 接入顺序

1. 确认 MiMo TTS 官方 API：URL、鉴权 header、模型名、音色 ID、价格。
2. 在现有 `backend/services/tts.py` 的 `mimo` provider 上做真实联调。
3. 增加 TTS 配置页或 `.env` 文档：`TTS_PROVIDER=mimo`、默认音色、主播 A/B 音色。
4. 为播客后台预生成音频，前端只播放已缓存 MP3。
5. 为追问回答增加短文本 TTS 缓存，避免重复合成。
6. 独立开本地 TTS spike，优先 CosyVoice，成功后再考虑替换云 API。
